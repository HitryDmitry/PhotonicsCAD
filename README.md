# Начало работы
Прежде всего необходимо установить Qt Creator и настроить бьютифайер. Это можно сделать так:

Preferences -> C++ -> Formatting mode -> выбираем Full formatting (по умолчанию подключится бьютифаейр ClangFormat) -> **Обязательно проверяем, что при сохранении код становится красивее.**
Можно также настроить специальный плагин, как это сделать описано [здесь](https://doc.qt.io/qtcreator/creator-beautifier.html), только нужно учитывать, что наша команта использует **только ClangFormat**.

# Разработка
При разработке мы следуем методологии [TDD](https://en.wikipedia.org/wiki/Test-driven_development) (Test-Driven Development). 

**Идея:** вместо того чтобы писать код, а потом проверять, работает ли он, процесс меняется на противоположный. 

Цикл разработки состоит из трех простых шагов (мантра «красный — зеленый — рефакторинг») 
1. ${\color{Red} Красный}$: Создается автоматизированный тест для новой функции. Поскольку самой функции еще нет, тест завершается неудачей (красный).
2. ${\color{Green} Зеленый}$: Пишется минимальный рабочий код, необходимый для успешного прохождения теста .
3. ${\color{Magenta} Рефакторинг}$: Код улучшается (становится чище и оптимизируется), при этом тест должен по-прежнему проходить успешно.

### **Что дает такой подход**
- Помогает заранее продумать архитектуру и требования
- Уменьшает количество ошибок (багов) в готовом приложении
- Гарантирует, что при добавлении новых фич старый код продолжает работать

# Основной тестовый фреймворк
Мы используем [doctest](https://github.com/doctest/doctest). Краткое руководство по работе с этим фреймворком приведено [здесь](https://github.com/doctest/doctest/blob/master/doc/markdown/tutorial.md).
